1. Go to [github.com/new](https://github.com/new)

## 🔐 Access Control & Permissions

### Set Up Teams

1. Go to Organization Settings → Teams
2. Create team: `iss-report-maintainers`
   - Members: Project lead, senior contributors
   - Permission: Maintain
3. Create team: `iss-report-writers`
   - Members: All section contributors
   - Permission: Write
4. Create team: `iss-report-viewers`
   - Members: Stakeholders
   - Permission: Read

### Branch Protection Rules

1. Go to Settings → Branches
2. Add rule for: `main`
3. Enable:
   - ✅ Require pull request reviews before merging (2 approvals)
   - ✅ Require status checks to pass before merging
   - ✅ Require branches to be up to date before merging
   - ✅ Require code reviews from code owners
   - ✅ Require conversation resolution before merging
   - ✅ Include administrators in restrictions
   - ✅ Restrict who can push to matching branches

## 📝 Repository Naming & Conventions

### Branch Naming Convention

```
main                          # Production/stable branch
develop                       # Integration branch (optional)

# Feature branches
feature/section-00-executive-summary
feature/section-06-circuit-design
feature/docs-update
feature/styling-improvements

# Bug fix branches
fix/compilation-error
fix/missing-reference

# Documentation branches
docs/readme-update
docs/contribution-guide

# Setup/configuration branches
setup/github-workflows
setup/ci-cd-pipeline
```

### Commit Message Convention

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Types:**
- `feat` - New feature/section
- `fix` - Bug fix
- `docs` - Documentation changes
- `style` - Formatting changes
- `refactor` - Code restructuring
- `perf` - Performance improvements
- `test` - Testing changes
- `chore` - Build/dependencies

**Scopes:**
```
section-06      - Section 06
appendix-a      - Appendix A
readme          - README file
github-config   - GitHub configuration
```

**Example:**
```
feat(section-06): Add circuit design section

Add comprehensive electrical system design documentation
including schematics, PCB layout, and BMS design.

Closes #45
```

## 🔄 GitHub Actions (CI/CD)

### Create Automated LaTeX Build

**File: `.github/workflows/latex-build.yml`**

```yaml
name: LaTeX Build & Validation

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main, develop ]

jobs:
  build:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Install LaTeX dependencies
      run: |
        sudo apt-get update
        sudo apt-get install -y \
          texlive-xetex \
          texlive-latex-extra \
          texlive-bibtex-extra \
          latexmk \
          biber
    
    - name: Build LaTeX document
      run: |
        cd /home/runner/work/iss-report/iss-report
        latexmk -xelatex -pdf main.tex
    
    - name: Upload PDF artifact
      uses: actions/upload-artifact@v3
      if: success()
      with:
        name: main.pdf
        path: main.pdf
    
    - name: Check for build errors
      if: failure()
      run: |
        echo "LaTeX build failed. Check logs above."
        exit 1
```

## 📊 GitHub Project Board Setup

### Create Automated Project

1. Go to Projects → New Project
2. **Name:** ISS Report - Section Completion
3. **Template:** Automated kanban
4. **Visibility:** Public

### Add Columns

- **Backlog** - Not started
- **In Progress** - Currently being worked on
- **In Review** - PR submitted, waiting for approval
- **Done** - Merged and complete

### Automate Workflow

**For each column, set automation:**
- **Backlog:** New issues → Backlog
- **In Progress:** Issues assigned → In Progress
- **In Review:** PR opened → In Review
- **Done:** PR merged → Done
