## 🔄 Workflow Guidelines

### Before Starting Work

1. **Update your local copy:**
   ```bash
   git pull origin main  # or your branch
   ```

2. **Identify your section(s):**
   - Assign yourself to a specific section from the table above
   - Document your assignment in the team log

3. **Create a working branch (if using git):**
   ```bash
   git checkout -b feature/section-XX-your-name
   ```

### During Writing/Editing

1. **Open the assigned section file:**
   ```bash
   # Example: working on Circuit Design section
   nano Section\ Files/06-circuit-design.tex
   ```

2. **Replace placeholder content:**
   - Find: `% Content placeholder`
   - Replace with your actual content

3. **Follow LaTeX conventions** (see section below)

4. **Keep related files near:**
   - Images in `images/` folder
   - References in `references.bib`

### After Completing Work

1. **Test compilation locally:**
   ```bash
   latexmk -xelatex -c && latexmk -xelatex main.tex
   ```

2. **Verify your section:**
   - Check generated `main.pdf`
   - Ensure your section has correct page numbers
   - Verify cross-references work

3. **Commit your changes (if using git):**
   ```bash
   git add Section\ Files/06-circuit-design.tex
   git commit -m "Add content to Section 06: Circuit Design & Electronics"
   git push origin feature/section-XX-your-name
   ```

4. **Create a Pull Request** for review before merging to main

## ✍️ Editing Conventions

### LaTeX Formatting Standards

#### Subsections
```latex
\subsection{Topic Name}
% Your content here
```

#### Emphasis
```latex
\textbf{Bold text}      % For strong emphasis
\textit{Italic text}    % For emphasis
\underline{Underlined}  % For underlines
```

#### Lists
```latex
% Unordered list
\begin{itemize}
    \item First item
    \item Second item
\end{itemize}

% Ordered list
\begin{enumerate}
    \item First step
    \item Second step
\end{enumerate}
```

#### Tables
```latex
\begin{table}[h]
    \centering
    \caption{Table Caption}
    \begin{tabular}{|c|c|c|}
        \hline
        Column 1 & Column 2 & Column 3 \\
        \hline
        Data 1 & Data 2 & Data 3 \\
        \hline
    \end{tabular}
\end{table}
```

#### Figures
```latex
\begin{figure}[h]
    \centering
    \includegraphics[width=0.8\textwidth]{images/your-image.png}
    \caption{Figure Caption}
    \label{fig:your-label}
\end{figure}
```

#### References
```latex
% Cite in text
According to \cite{author2023}, ...

% Add to references.bib
@article{author2023,
    author = {Author, A.},
    title = {Article Title},
    journal = {Journal Name},
    year = {2023}
}
```

#### Special Characters
```latex
\&      % Ampersand
\%      % Percent
\$      % Dollar sign
\{      % Left brace
\}      % Right brace
\textasciitilde  % Tilde
```

### Content Conventions

- **Line length:** Keep lines under 80 characters for readability
- **Spacing:** One blank line between sections
- **Comments:** Use `%` for inline comments, `%...%` for block comments
- **Indentation:** Use 4 spaces or consistent tabs
- **Language:** Professional English, past/present tense for technical content
- **Consistency:** Match terminology across all sections (create a glossary reference)

## 🔨 Building & Compilation

### Requirements

```bash
# Check if you have these installed:
which xelatex      # XeLaTeX engine
which latexmk       # Build manager
which biber         # Bibliography processor
```

### Building the Document

**Full clean build:**
```bash
cd /home/yassine/Desktop/ISS_Report
latexmk -xelatex -c              # Clean
latexmk -xelatex main.tex        # Build
```

**Quick rebuild:**
```bash
latexmk -xelatex main.tex
```

**Watch mode (auto-rebuild on file changes):**
```bash
latexmk -xelatex -pvc main.tex
```

**Output:** `main.pdf`

### Common Issues & Solutions

| Issue | Solution |
|-------|----------|
| `fontspec` error | Ensure using XeLaTeX, not pdfLaTeX |
| Missing references | Run `biber main` and recompile |
| Undefined citations | Add entries to `references.bib` |
| Image not found | Check path and file exists in `images/` |
| Table/Figure numbering wrong | Recompile 2-3 times |

## 👥 Contribution Process

### Step-by-Step for Contributors

1. **Check available sections** in the project status document
2. **Assign yourself** to a section (update team log)
3. **Create feature branch:**
   ```bash
   git checkout -b feature/section-NN-your-name
   ```
4. **Edit your section file** following conventions above
5. **Build and test locally:**
   ```bash
   latexmk -xelatex -c && latexmk -xelatex main.tex
   ```
6. **Review in PDF** - does it look correct?
7. **Commit with clear message:**
   ```bash
   git commit -m "Add Section XX: Descriptive Title"
   ```
8. **Push to remote:**
   ```bash
   git push origin feature/section-NN-your-name
   ```
9. **Create Pull Request** on GitHub/GitLab
10. **Wait for review** and address feedback
11. **Merge to main** after approval

### Code Review Checklist

Before submitting for review, verify:

- ✅ Section placeholder replaced with actual content
- ✅ All subsections have content (not empty)
- ✅ LaTeX compiles without errors
- ✅ No undefined references or citations
- ✅ Images are properly referenced and located
- ✅ Spelling and grammar checked
- ✅ Formatting consistent with other sections
- ✅ No hardcoded page numbers or section numbers
- ✅ Cross-references use labels (`\label{}`, `\ref{}`)
- ✅ Bibliography entries properly formatted

## 💡 Best Practices

### Document Management

1. **Don't edit `main.tex` unless necessary** - only add new section includes
2. **Keep sections independent** - avoid strong dependencies between sections
3. **Use consistent file naming** - follow `NN-section-name.tex` pattern
4. **Create meaningful commit messages** - help others understand changes

### Writing Quality

1. **Write clearly and concisely** - avoid jargon where possible
2. **Use active voice** - "The system processes data" not "Data is processed"
3. **Consistency is key** - define terms once, use consistently
4. **Proofread before submitting** - use spell-check
5. **Cite sources properly** - maintain `references.bib` accurately

### Collaboration

1. **Communicate with team** - announce what you're working on
2. **Ask for help early** - don't struggle alone
3. **Review others' work** - provide constructive feedback
4. **Respect deadlines** - plan your section completion time
5. **Use pull requests** - even for small changes, maintain version control

### Performance

1. **Keep image file sizes reasonable** - compress before including
2. **Avoid overly complex tables** - break into smaller, readable tables
3. **Don't duplicate content** - reference existing sections instead
4. **Clean up build artifacts** - run `latexmk -c` before committing

### Common Mistakes to Avoid

❌ **Don't:** Edit `main.tex` without coordinating with team
❌ **Don't:** Mix spaces and tabs in indentation
❌ **Don't:** Include build artifacts (`.aux`, `.log`, etc.) in commits
❌ **Don't:** Use special characters without escaping (`$`, `&`, `%`, etc.)
❌ **Don't:** Forget to add images to `images/` folder
❌ **Don't:** Leave `% Content placeholder` comments in final content
❌ **Don't:** Create broken cross-references with wrong `\label{}` names

✅ **Do:** Communicate with team about section assignments
✅ **Do:** Test compilation after each major edit
✅ **Do:** Follow LaTeX conventions and formatting
✅ **Do:** Write meaningful commit messages
✅ **Do:** Review the PDF output carefully
✅ **Do:** Ask questions if unsure about conventions
✅ **Do:** Keep sections focused and well-organized
